# 从 OpenClaw 到 Hermes Agent

## 一、导语

最近一段时间，Hermes Agent 在 X 和 GitHub 上冒得很快。它火，不只是因为 star 涨得凶，更因为不少 OpenClaw 老用户用了之后觉得“终于省事了”。以前升级 OpenClaw，常常要预留两三个小时排错；换到 Hermes，有人从 0.5 升到 0.8 直接跑通。对普通用户来说，这比“自进化”之类的大词更有说服力：少折腾、少翻车、能一直在线，才是真体验。

![X 上关于 Hermes 升级体验的讨论](./assets/6E84E04BF785493F4A6BF3B209A591E4.jpg)

## 二、Hermes Agent 是什么

如果把大模型看成一匹马，Hermes 更像那套把马、人和路线连在一起的马具。模型负责思考，Hermes 负责把任务拆开、调用工具、保存记忆、约束动作，再把结果接回到下一步。它的特别之处在于不只“干一次活”，而是会把做过的事慢慢沉淀下来：记住你的偏好，保留跨会话上下文，把有效做法整理成 skill。你可以把它装在本地电脑、Mac mini 或便宜 VPS 上，接 Telegram 之类的消息入口，当成一个长期在线的个人助理来用。

![用户关于 Hermes 持续执行和省 token 的反馈](./assets/C479815ADCD7AA6F4E7B94F24FE83678.jpg)

## 三、OpenClaw 和 Hermes 有什么区别

OpenClaw 的长处是生态大、工具多、接入广，适合喜欢折腾、需要现成插件的人；但它的问题也很现实：更新后容易冒回归 bug，长任务容易跑偏，插件和安全边界也让维护成本越来越高。Hermes 走的是另一条路，功能没那么铺张，重点放在稳定、记忆和持续改进上。很多人转过去，不是为了追新概念，而是因为它遇到报错更愿意继续试，长对话 token 涨得没那么快，重开之后也还能接上之前的习惯。简单说，OpenClaw 更像一套热闹但难养的工具箱，Hermes 更像一个越用越顺手的常驻助理。要是你最在意少出错、少维护、迁过去就能接着干活，Hermes 更合适；要是你重度依赖现成生态，OpenClaw 也还没到完全退场的时候。
