## 最近这匹「马」为什么火了

最近几周刷 X 和 GitHub，一个名字反复出现：Hermes Agent。

2 月底开源，首月 2.2 万星。4 月 8 日 v0.8.0 发布后单日涨了 6400 颗，到现在总数突破 4.7 万，连续多天霸占全球开源趋势榜榜首。数字背后是真金白银的迁移潮——大量 OpenClaw 用户正在把自己的 Agent 搬家到 Hermes。

它是什么？一句话说：一个会「长本事」的 AI 智能体。做完任务会自动总结经验、沉淀成可复用的技能，下次再碰到类似情况直接调用，越用越顺手。部署方式从 5 美元 VPS 到 Docker 到 Serverless 都有，兼容 200 多个大模型，Telegram、Discord、Slack 全平台接入，一条命令就能装好。

有人说它是 OpenClaw 的平替，有人说根本不是同一个物种。在 X 上争来争去，但有一件事倒是没什么争议——试过的人很少回头。

![Gaj Varma 与 Teknium 的推文](./assets/6E84E04BF785493F4A6BF3B209A591E4.jpg)

用户 Gaj Varma 在 X 上说，从 0.5 更新到 0.8，要是 OpenClaw 得花两三个小时修问题，Hermes 直接跑通了——695 个 commit，零故障。Hermes 创始人 Teknium 在底下回了一句：「我倒想问 OpenClaw 到底干了啥，才让升级这么折腾。」

![林亦 LYi 的推文](./assets/C479815ADCD7AA6F4E7B94F24FE83678.jpg)

国内开发者林亦也给了实测反馈：Hermes 碰到报错不会莫名其妙停下来，会一直尝试到有明确结果才罢休；而且省 token——聊很久上下文也就维持在三四万 token，换 OpenClaw 早就飙过十万了。

「准备放弃龙虾转爱马仕了，龙虾记忆太差了，爱马仕无论怎么重开、过多久都能记住，太香了！」社区里这类声音越来越密集。有用户把 Hermes 装在一台始终在线的 Mac mini 上，通过 SSH 和 Telegram 随时操控，日历、Gmail、Todoist 的信息先汇总分析，再分派给 Claude Code 执行，结果同步回 Obsidian vault——全程无人值守。

还有人 2.5 小时用 Hermes 做出了《百战天虫》克隆版，Agent 自己把物理引擎逻辑整理成了可复用的 skill 插件。

这匹马确实在跑，而且跑得不慢。

## Hermes Agent 到底在做什么不一样的事

先说一个最近行业里频繁冒出来的概念：**Harness**。

字面意思是马具。在 AI 语境里，它指的是套在模型身上、连接人类需求的那层控制框架。Agent 的「大脑」是模型本身——比如 Claude Opus 4.6 或者 Gemini——而大脑之外的一切，调度逻辑、记忆管理、权限控制、任务编排，统统属于 Harness 的管辖范围。

它不让马更聪明，但决定了这匹马往哪跑、跑多快、什么时候该停。

这个判断不是某一家的自说自话。今年 2 月 OpenAI 发了篇博客专门讲 Harness Engineering，论证三个工程师靠一套好的 Harness 就能让 Agent 在五个月里写出百万行代码。行业共识正在快速收敛到一个点上：Prompt Engineering 和 Context Engineering 都不够用了，得有更高一层的约束系统。

用约束换自主——听起来反直觉，但道理很朴素：越自主的东西越容易跑偏，马具越好，马反而跑得越远。

Hermes Agent 就是 Nous Research 在这个方向上交出的答卷。

Nous Research 不是野鸡团队。这家自称「去中心化 AI 研究实验室」的公司，同时在推进一个叫 Psyche 的去中心化训练网络——试图用区块链协调全球闲置 GPU 来训练模型。Hermes 是他们整个去中心化 AI 基础设施里最贴近用户的那层入口。MIT 许可，完全免费开源。

和传统 Agent 的根本区别在于：Hermes 内置了一套完整的学习闭环。

完成复杂任务后，它会自动从执行过程中抽象方法论，沉淀为可复用的 Skill，下次碰到类似问题直接调用并持续迭代优化。这意味着它的能力边界不是写死的，而是在使用中逐渐扩展的——网站结构变了，它自己调整抓取逻辑；用过的解决方案出了更好的版本，它自己替换。

记忆系统是分层的。除了显性的信息存储，最核心的是它在构建一个「关于你的模型」——跨会话的交互中，它逐渐理解你的代码风格、技术栈偏好、对报错的容忍度，甚至会定期自行整理和固化这些认知。

Hermes 官方那句口号「the agent that grows with you」并不只是营销话术，它指向的是一整套围绕学习闭环构建的底层架构：解决问题 → 记录经验 → 复用经验 → 优化方法 → 反哺自身。当一个 Agent 真正跑通这个循环，它就不再是一个工具，而更接近一种可持续积累的资产。

数据、记忆、行为轨迹都留在你自己的机器上，不上传到任何第三方云端。AI 的能力不再只是被租用，而是可以被拥有。

当然，路径还很初期。记忆的噪音、技能的质量、训练闭环的稳定性都是待打磨的问题，离普通用户无感使用还有距离。但方向很清晰——以后评价一个 Agent，可能不再是看它「现在能做什么」，而是看它「用了半年之后变成了什么」。

## 龙虾 vs 爱马仕：两条路线之争

在聊差异之前，先说说它们的共同底色：**数字主权**。

不管是 Hermes 还是龙虾，出发点都一样——数据不上传到不可控的云端，记忆和执行过程留在本地或私有环境；交互走 Telegram、Discord 这类即时通讯渠道，不依赖笨重的 Web UI；支持 24/7 后台运行，不需要人时刻盯着。它们本质上都在解决同一个问题：让 AI 真正归你所有。

但在这层共识之下，两者走向了几乎相反的方向。

困扰养虾人最久的两件事：**Token 太贵，长程任务不靠谱。**

把 OpenClaw 丢进一个超过 24 小时的长任务里不管，大概率等来两种结果——Token 烧完事情只做了一半，或者某步走偏后一路错到底，交出来的东西没法用。

![Summer Yue 的推文](./assets/aba5968c-ca5c-4e95-a320-7d125ebe36eb.png)

Meta 安全对齐总监 Summer Yue 的故事至今在圈子里流传：她明明给 OpenClaw 设了「操作前确认」，结果 Agent 一路狂飙删完了收件箱。手机上根本拦不住，最后冲到 Mac mini 前面才止住——她自己说「像在拆炸弹」。

这不是孤例。OpenClaw 因为病毒式增长，代码库和 ClawHub 插件生态膨胀太快，质量控制跟不上。2026 年初至今已披露多个高危漏洞：WebSocket 劫持令牌的一键远程代码执行（CVE-2026-25253，CVSS 8.8）、命令注入、路径遍历、权限提升……ClawHub 里甚至被安全研究机构发现了数百个恶意插件，偷取凭证或植入窃取器。更新后「25% 概率炸掉某个功能」已经是用户间心照不宣的默认预期。

Hermes 在这些具体的痛点上确实好过不少——代码库精炼，没有 ClawHub 那种海量第三方插件的供应链风险；危险操作必须人工批准，内置 Tirith 预执行扫描器；到目前为止没出现过集中的高危 CVE 或供应链攻击事件。

但更根本的差异在底层哲学。

**技能获取**：OpenClaw 的能力边界由人类预设——开发者写好 Skill，Agent 按部就班执行，稳定可控。Hermes 的技能从任务经验中涌现——做完一件事后自动提炼方法论，下次直接用，能力边界在使用中自己长出来。

**记忆机制**：OpenClaw 本质是 RAG 思路，知道信息存在哪，需要时去查。Hermes 则在构建一个「关于你的模型」——它记住的不是一条条目，而是你这个人的工作方式。跨会话积累之后，它知道你偏好 TypeScript 还是 Python，遇到报错是倾向于直接重写还是逐步排查。

**适用场景**：如果任务容错率极低、步骤明确（比如金融数据处理），OpenClaw 的权限管控更严格、行为更可预测。如果是探索性开发、流程模糊、需要不断试错的复杂工程，Hermes 的自主性和长期记忆会帮你省下不少来回沟通的成本。

社区的共识不是「谁替代谁」。Reddit 和 X 上比较主流的用法是让 Hermes 当高级规划器，跑在 OpenClaw 的工具层之上：龙虾负责干活，爱马仕负责动脑。Hermes 官方也直接提供了 `hermes claw migrate` 命令，一行就能把已有的 OpenClaw 技能、记忆和配置搬过来。

也有更务实的看法：「我不会直接换掉 Claude Code 的工作流，但把 Hermes 挂在旁边专门做常在线消息处理和个人助理，确实有吸引力。」

不管你选哪种方式，至少可以肯定一点：龙虾不再是唯一的选项了。
